\# TaskPlanner — система планирования задач ИТ-отдела с контролем нагрузки



Приложение для управления задачами, проектами, спринтами и сотрудниками.

Реализует ролевую модель (работник, менеджер, администратор), канбан-доску, матрицу Эйзенхауэра,

календарь дедлайнов, отчёты (burndown, точность оценок, нагрузка) и экспорт в PDF/Excel.



\## Основные возможности



\- Аутентификация по JWT с разграничением прав (Worker, Project Manager, Admin)

\- Канбан-доска с перетаскиванием задач и двойным кликом для деталей

\- Матрица Эйзенхауэра и календарь дедлайнов

\- Отчёты и диаграммы: нагрузка команды, просроченные задачи, burndown, точность оценок,

&#x20; прогресс проектов, топ сотрудников по трудозатратам

\- Экспорт отчётов в PDF и Excel

\- Автоматический учёт трудозатрат по истории изменения статусов

\- Контроль перегрузки сотрудников (запрет назначения задач при загрузке >150%)

\- Заметки к задачам

\- Поддержка производственного календаря (настраиваемые выходные)

\- Material Design 3 (светлая тема DeepPurple/Lime) на клиенте



\## Технологический стек



\- Клиент: C# 10, WPF (.NET 10), MaterialDesignInXAML, LiveCharts2, Newtonsoft.Json

\- Сервер: Node.js, Express.js, pg (драйвер PostgreSQL), bcrypt, jsonwebtoken

\- База данных: PostgreSQL

\- Тестирование: xUnit (юнит-тест конвертера BooleanToVisibility)

\## Быстрый старт (развёртывание для демонстрации)



1\. \*\*База данных (PostgreSQL)\*\*  

&#x20;  Убедитесь, что PostgreSQL запущен (порт 5432). Создайте пустую базу `taskplanner`.  

&#x20;  Выполните скрипт инициализации:  

&#x20;  `psql -U postgres -d taskplanner -f database/full\_init.sql`  

&#x20;  После этого база будет наполнена демо-данными.



2\. \*\*Сервер (Node.js)\*\*  

&#x20;  Перейдите в папку `server/`. Установите зависимости: `npm install`  

&#x20;  Проверьте настройки подключения к БД в `config/db.js` (пользователь, пароль).  

&#x20;  Запустите сервер: `npm start` (или `node index.js`)  

&#x20;  Сервер слушает порт 3001.



3\. \*\*Клиент (WPF)\*\*  

&#x20;  Откройте `client/TaskPlannerClient.sln` в Visual Studio (или Rider).  

&#x20;  Убедитесь, что все NuGet-пакеты восстановлены.  

&#x20;  Проверьте URL API в `appsettings.json` или в `Services/ApiService.cs`  

&#x20;  (по умолчанию `http://localhost:3001/api`).  

&#x20;  Запустите приложение (F5).



4\. \*\*Вход\*\*  

&#x20;  Используйте одну из тестовых учётных записей:



&#x20;  - Администратор: `admin` / `admin`

&#x20;  - Менеджер проектов: `manager` / `manager`

&#x20;  - Работник (Иванов): `ivanov` / `worker`

&#x20;  - Работник (Петрова): `petrova` / `worker`

&#x20;  - Другие учётные записи см. в таблице Users в базе



&#x20;  Все пароли хранятся в виде bcrypt-хэшей.



\## Сценарии демонстрации



\- Worker заходит под `ivanov`, видит канбан-доску со своими задачами, меняет статусы,

&#x20; открывает матрицу и календарь.

\- Manager заходит под `manager`, создаёт проект/спринт, управляет задачами,

&#x20; открывает отчёты (нагрузка, burndown, точность оценок), экспортирует их в Excel.

\- Admin заходит под `admin`, управляет сотрудниками, командами, пользователями, удаляет проекты.
