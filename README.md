# TaskPlanner — система планирования задач ИТ-отдела с контролем нагрузки



Приложение для управления задачами, проектами, спринтами и сотрудниками.

Реализует ролевую модель (работник, менеджер, администратор), канбан-доску, матрицу Эйзенхауэра,

календарь дедлайнов, отчёты (burndown, точность оценок, нагрузка) и экспорт в PDF/Excel.



## Основные возможности



- Аутентификация по JWT с разграничением прав (Worker, Project Manager, Admin)

- Канбан-доска с перетаскиванием задач и двойным кликом для деталей

- Матрица Эйзенхауэра и календарь дедлайнов

- Отчёты и диаграммы: нагрузка команды, просроченные задачи, burndown, точность оценок,

- прогресс проектов, топ сотрудников по трудозатратам

- Экспорт отчётов в PDF и Excel

- Автоматический учёт трудозатрат по истории изменения статусов

- Контроль перегрузки сотрудников (запрет назначения задач при загрузке >150%)

- Заметки к задачам

- Поддержка производственного календаря (настраиваемые выходные)

- Material Design 3 (светлая тема DeepPurple/Lime) на клиенте



## Технологический стек



- Клиент: C# 10, WPF (.NET 10), MaterialDesignInXAML, LiveCharts2, Newtonsoft.Json

- Сервер: Node.js, Express.js, pg (драйвер PostgreSQL), bcrypt, jsonwebtoken

- База данных: PostgreSQL

- Тестирование: xUnit (юнит-тест конвертера BooleanToVisibility)

## Быстрый старт

### Через Docker (рекомендуется)

1. Установите Docker Desktop.
2. Клонируйте репозиторий.
3. В корне проекта выполните:
   ```bash
   docker compose up -d
4. Сервер будет доступен на http://localhost:3001.

5. Клиент (WPF) откройте в Visual Studio, убедитесь, что URL API указывает на http://localhost:3001/api, и запустите (F5).

### Без Docker (ручная установка)
1. База данных (PostgreSQL)
Убедитесь, что PostgreSQL запущен (порт 5432). Создайте пустую базу taskplanner.
Выполните скрипт инициализации:
psql -U postgres -d taskplanner -f database/full_init.sql
После этого база будет наполнена демо-данными.

2. Сервер (Node.js)
Перейдите в папку server/. Установите зависимости: npm install
Проверьте настройки подключения к БД в config/db.js (пользователь, пароль).
Запустите сервер: npm start (или node index.js)
Сервер слушает порт 3001.

3. Клиент (WPF)
Откройте client/TaskPlannerClient.sln в Visual Studio (или Rider).
Убедитесь, что все NuGet-пакеты восстановлены.
Проверьте URL API в appsettings.json или в Services/ApiService.cs
(по умолчанию http://localhost:3001/api).
Запустите приложение (F5).



4. **Вход**  

Используйте одну из тестовых учётных записей:



- Администратор: `admin` / `admin`

- Менеджер проектов: `manager` / `manager`

- Работник (Иванов): `ivanov` / `worker`

- Работник (Петрова): `petrova` / `worker`

- Другие учётные записи см. в таблице Users в базе



Все пароли хранятся в виде bcrypt-хэшей.



## Сценарии демонстрации



- Worker заходит под `ivanov`, видит канбан-доску со своими задачами, меняет статусы, открывает матрицу и календарь.

- Manager заходит под `manager`, создаёт проект/спринт, управляет задачами, открывает отчёты (нагрузка, burndown, точность оценок), экспортирует их в Excel.

- Admin заходит под `admin`, управляет сотрудниками, командами, пользователями, удаляет проекты.
